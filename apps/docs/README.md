# @felix/docs

The public Felix docs site — [Starlight](https://starlight.astro.build) building a fully static
site, deployed as a Cloudflare Workers static-assets Worker at `docs.felix.run`.

**No prose lives here.** Markdown stays co-located with the package it documents:

- `packages/harness/docs/` — README (site index), `guide/`, `internals/`
- `packages/commerce/docs/` — the Commerce section

`scripts/sync-content.ts` aggregates those sources into `src/content/docs/` (gitignored) before
every dev/build: it injects Starlight frontmatter (title from the first `# H1`, sidebar order from
the narrative order lists) and rewrites relative markdown links to site routes (links escaping the
docs trees go to GitHub). Edit the sources, never the synced copies.

The API reference is **not** built here — it is runtime-generated by the API Worker (`apps/api`;
Scalar over `/openapi.json` at `make.felix.run/docs`); the sidebar links to it.

**Design system**: every Felix surface themes itself from one workspace package,
**`@felix/design`** (`packages/design/src/tokens.ts` — neutral monochrome, light + dark). The API
Worker renders it onto Scalar's CSS variables, the sync script renders it into this site's
`src/styles/theme.css` (gitignored, regenerated every build), and the commerce storefront widget
takes its neutral chrome from the same scale. Change a hex in the token module and every surface
moves together.

```bash
pnpm --filter @felix/docs dev      # sync + astro dev
pnpm --filter @felix/docs build    # sync + astro build → dist/
pnpm --filter @felix/docs deploy   # build + wrangler deploy (static assets)
```

Root shortcuts: `pnpm docs:dev` / `docs:build` / `docs:deploy`.
